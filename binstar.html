Setting up a machine to be a binstar builder¶

  1. Install binstar-build

    conda install -c https://conda.binstar.org/binstar binstar-build

  2. Create an authentication token to binstar

    binstar auth –create –name build-workers –scopes api:build-worker > ~/binstar.token

  3. Run the worker

    binstar-build -t $(<~/binstar.token) worker builder -u Nikea

Configuring a conda package for binstar job submission¶

  1. Clone the conda-prescriptions repo from https://github.com/NSLS-II/conda-prescriptions

  2. init a binstar build if one doesn’t exist already

    binstar-build init

  3. Edit the binstar-build file .binstar.yml so that it basically looks like this

    package: package_name user: binstar_user_name # place to upload the file to platform:

    • linux-64
    • osx-64
    engine:
    • python=2.7
    • python=3.4
    install:
    • conda config –add channels Nikea
    script:
    • conda build .

    build_targets: conda iotimeout: 600

# Submit the binstar build with

binstar-build submit

Submitting builds to binstar¶

  1. Login to your binstar

    binstar config –set url https://conda.nsls2.bnl.gov/api binstar login

  2. Configure your binstar account to use Nikea/builders to be the default queue

    1. go to https://conda.nsls2.bnl.gov
    2. go to User Settings (top left)
    3. go to Build Queues
    4. Set the default queue to be build/Nikea/builders

  3. Submit the build

    binstar-build submit
